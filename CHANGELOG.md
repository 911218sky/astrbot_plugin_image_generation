# 更新日誌

## v1.0.15 - 2026-07-19

- 放寬供應商 JSON 回應封裝上限至 48 MiB，支援較大的 2K/4K Base64 圖片回應，並接受 UTF-8 BOM。
- JSON 格式或回應過大不再重試相同請求，避免批量任務因固定格式問題重複等待。
- 插件頁改為每 2 秒更新進行中任務，顯示批次完成數、成功/失敗數、目前階段、供應商耗時與總耗時。

## v1.0.14 - 2026-07-19

- 模型切換會等待進行中的生圖請求完成，避免關閉仍在使用的供應商連線。
- 模型切換後重新套用 LLM 生圖工具能力欄位，避免沿用舊模型的寬高比、解析度或參考圖參數。
- `enable_llm_tool` 遇到非布林值時回退到安全預設，且工具 schema 的批量上限會跟隨實際配置。
- 強化 LLM 工具對非字串提示詞與尺寸參數的邊界處理。
- 明確的 4xx/API Key 錯誤不再重試，Gemini 路徑也會保留 HTTP 狀態碼，降低無效等待與供應商請求量。
- 模型切換期間排隊中的生成任務會等待新適配器完成，不再因短暫切換狀態誤判未初始化。
- 補充 LLM 工具調用流程、批量併發與速度影響因素的文件。

## v1.0.13 - 2026-07-19

- 所有供應商回傳的圖片 URL 統一使用安全下載器，限制公開網路位址、重新導向與回應大小。
- 限制供應商 JSON、Base64 與最終圖片輸出大小，避免異常回應造成記憶體或磁碟耗盡。
- 強化直接批量呼叫與設定檔數字解析，並在 `/img gen` 缺少 API Key 時立即回報。
- 修正測試套件從插件目錄直接執行時的匯入路徑。

## v1.0.12 - 2026-07-19

- 保留 `generation.show_task_started` 通用開關，控制 `/img gen` 與 AI 生圖工具是否傳送任務啟動提示；關閉時仍會建立背景任務。
- 合併 hardening 分支，加入安全參考圖收集、佇列 admission 與原子圖片發布流程。
- 新增 `count` 批量生圖：跨供應商 fan-out、最多四路並行、部分成功回報與按成功圖片數扣除額度。
- 新增 `generation.max_batch_count` 設定，並修正批量輸出檔名碰撞。
- 修正背景任務例外未消費、定時任務替換時誤刪新任務，以及即夢積分任務 HTTP 會話未關閉的問題。

## 未發布

## v1.0.11 - 2026-07-17

- 新增 `max_queued_tasks` 與統一 admission 流程；容量已滿時會在下載參考圖或呼叫供應商前拒絕。
- 強化安全稽核輸出的暫存、整批發布、回滾與殘留清理；無法完成安全清理時採取封閉式失敗。
- 本地參考圖只允許插件資料目錄、AstrBot `data/temp` 與 `data/attachments`，並以描述符逐層開啟及驗證檔案身分。
- 遠端參考圖逐次驗證公開 DNS、實際連線端點與重新導向，拒絕私有位址及 DNS rebinding，並限制連線數、逾時、重新導向及下載大小。
- 照片快取頁只列出已成功發布的圖片，排除稽核中、已封鎖及清理殘留檔案。
- 修正取消傳播、遠端資源釋放、設定邊界及供應商與模型重新載入狀態，提升失敗與重載流程可靠性。

## v1.0.10 - 2026-06-07

- Dashboard 自動刷新間隔由 5 秒調整為 30 秒，降低 WebUI 與插件 API 壓力。
- 手動重新整理仍會立即更新總覽與目前照片快取頁。

## v1.0.9 - 2026-06-07

- 照片快取頁改為分頁載入，可選每頁 12/24/36 張。
- 新增 `page/images` 分頁 API，只為目前頁面的圖片產生縮圖。
- 總覽自動刷新不再附帶快取圖片縮圖，避免快取很多時重複載入大量圖片。

## v1.0.8 - 2026-06-07

- 照片快取頁新增原圖放大預覽與下載功能。
- 生成圖片現在會寫入插件自己的 `image_metadata.json`，記錄檔名、任務 ID、模型、提示詞、會話與生成狀態。
- 快取圖片卡片與預覽視窗會顯示生成模型、提示詞摘要與任務資訊；舊圖片沒有 metadata 時會標示為未記錄。
- 新增 `core/image_metadata_store.py`，把圖片 metadata 儲存邏輯從主流程與 Page API 中拆出，方便維護。

## v1.0.7 - 2026-06-07

- 新增 AstrBot 官方插件 Pages 看板，可查看排隊中、生成中、最近完成/失敗紀錄與照片快取。
- 生圖任務現在會區分 `queued`、`running`、`success`、`failed`、`blocked`，方便 Web 頁面與 `/img tasks` 觀察。
- 新增 `core/page_api.py`，以插件執行期快照提供看板資料，不寫入 AstrBot 核心資料庫。
- `requirements.txt` 加上依賴上限，降低未來大版本依賴破壞風險。

## v1.0.6 - 2026-06-07

- 參考 LivingMemory 的指令組寫法，將生圖、任務、模型與預設管理統一整理到 `/img` 指令組。
- 新增 `/img gen`、`/img tasks`、`/img model`、`/img preset`、`/img help`，避免管理指令分散在 `/img_model`、`/img_tasks`、`/preset`。

## v1.0.5 - 2026-05-08

- LLM Tool 啟動圖生圖任務時，通知訊息現在會顯示參考圖數量。

## v1.0.4 - 2026-05-07

- OpenAI 生圖流程改為僅保留 GPT Image 系列
- 移除內建預設提示詞
- 補齊繁體中文介面、提示訊息與設定說明
- 強化 LLM Tool 說明與系統提示，讓模型更容易自動呼叫生圖工具
- LLM Tool 現在會依自然語言內容主動判斷較合適的寬高比與解析度
- 新增生成開始通知與失敗通知
- 補強請求異常時的失敗回覆，例如 `Server disconnected` 也會通知使用者
- 新增 `/img_tasks` 指令，可查看目前正在進行中的生圖任務
- 新增成功耗時提示開關，可在 Web 設定中控制
- 重試次數改為可設定，最高 5 次，並補充終端日誌提示
- 外掛識別名稱改為 `astrbot_plugin_image_generation_911218sky`，避免與原作者同名外掛發生更新比對衝突
- 新增會話黑名單
- 新增稽核白名單會話
- 新增安全稽核功能
- 新增 Grok AI 適配器
