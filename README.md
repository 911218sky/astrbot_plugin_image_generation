# 通用生圖插件

![:name](https://count.getloli.com/@astrbot_plugin_image_generation?name=astrbot_plugin_image_generation&theme=miku&padding=7&offset=0&align=top&scale=1&pixelated=1&darkmode=auto)

### 支援多種主流生圖接口的通用圖像生成插件，支援文生圖、圖生圖、LLM 自動呼叫，以及可自訂的預設系統。

> 外掛識別名稱：`astrbot_plugin_image_generation_911218sky`

## 致謝

本插件基於原作者 [Railgun19457](https://github.com/railgun19457) 的
[`astrbot_plugin_image_generation`](https://github.com/railgun19457/astrbot_plugin_image_generation)
進行修改與延伸，原始設計、整體架構與多適配器生圖能力都來自原作者的工作。

## 這個版本的調整

相較於原版，這個版本主要做了以下修改：

- 將插件內大部分對外文字、提示訊息與設定說明調整為繁體中文
- OpenAI 生圖流程改為只保留 GPT Image 系列，不再保留 DALL-E 分支
- 強化 LLM Tool 的說明與系統提示，讓模型更容易在有生圖意圖時自動呼叫工具
- 調整 Telegram 指令描述與指令刷新流程，讓 `/` 指令提示更容易正確出現
- 加入可由使用者設定、且上限為 5 次的重試機制與終端日誌提示，提升失敗場景的可觀察性
- 移除內建預設提示詞，改為由使用者自行維護預設內容
- 參考 LivingMemory 的指令組寫法，將生圖、任務、模型與預設管理統一整理到 `/img` 指令組

***

### 命令
- `/img gen <提示詞或預設名稱> [額外提示詞]`
  - 生成圖片。示例: `/img gen 一隻可愛的小貓`
  - 使用預設。示例：`/img gen 動漫風`
  - 使用預設並附加額外提示詞。示例: `/img gen 動漫風 藍色髮`
  - **圖生圖模式**：如果訊息中包含圖片、引用包含圖片的訊息，或 @ 使用者，將自動作為參考圖進入圖生圖模式（需適配器支援）。
  - @ 使用者時會取得其頭像作為參考圖。示例: `/img gen 像素風 @使用者A`
  - 也可使用 `/img generate`、`/img create`、`/img draw` 作為生圖別名。

- 可在提示詞中直接加入 `@self`，將當前 Bot 的頭像作為參考圖。示例：`/img gen 像素風 @self`

- `/img model`
  - 顯示可用模型列表和當前使用的模型。模型格式為 `供應商名稱/模型名稱`。
  - `/img model <序號>` - 切換到指定序號的模型。

- `/img tasks`
  - 查看目前正在進行中的生圖任務。
  - 會顯示任務 ID、任務類型、執行秒數、比例、解析度與提示詞摘要。

- `/img preset`
  - 顯示目前所有預設。
  - `/img preset add <預設名:預設內容>` - 新增預設。
  - `/img preset del <預設名>` - 刪除指定預設。

- `/img help`
  - 顯示 `/img` 指令組說明。

### 功能特性

- ✅ **多供應商配置**：支援配置多個生圖服務供應商，每個供應商可擁有獨立的 API Key、Base URL 及模型列表。
- ✅ **模型動態切換**：透過指令即時檢視和在不同供應商的模型間切換。
- ✅ **供應商/模型體系**：採用 `供應商名稱/模型名稱` 的統一標識體系。
- ✅ **文生圖**：根據文字描述生成圖片。
- ✅ **圖生圖**：基於參考圖片（支援多張）生成新圖片，支援自動識別訊息圖片、引用圖片及 @ 使用者頭像。
- ✅ **LLM 整合**：支援作為工具被 LLM 自動呼叫，並可依自然語言需求主動判斷較合適的寬高比與解析度。
- ✅ **進階預設系統**：支援自訂提示詞模板，預設內容支援 JSON 格式以配置特定比例和解析度。
- ✅ **多 API Key 輪詢**：支援配置多個 API Key，自動輪詢與錯誤重試（次數可設定，最高 5 次），提高穩定性。
- ✅ **完善的控制**：支援併發任務限制、使用者頻率限制（Rate Limit）及每日生圖數量限制。

| `gemini_openai` |   ✅    |   ✅    |   ❌    |   ❌    | OpenAI 相容格式的 Gemini 接口                                         |
| `openai`        |   ✅    |   ✅    |   ❌    |   ✅    | OpenAI GPT Image 系列（gpt-image-1、gpt-image-1-mini、gpt-image-1.5 等） |
| `z_image_gitee` |   ✅    |   ❌    |   ✅    |   ✅    | Gitee AI (z-image-turbo)                                              |
| `jimeng2api`    |   ✅    |   ✅    |   ✅    |   ✅    | 適用於[iptag/jimeng-api](https://github.com/iptag/jimeng-api)的適配器 |

#### 特殊說明

### 配置項

#### 基礎配置
| 配置項            | 型別    | 預設值 | 說明                                  |
| :---------------- | :------ | :----- | :------------------------------------ |
| `enable_llm_tool` | boolean | `true` | 是否允許 LLM 在對話中自動呼叫生圖功能 |

#### API 供應商配置 (`api_providers`)
支援配置多個供應商，每個供應商項包含：
| 配置項             | 型別   | 說明                                                                 |
| :----------------- | :----- | :------------------------------------------------------------------- |
| `name`             | string | 適配器名稱，用於區分不同供應商。生圖模型格式為 `適配器名稱/模型名稱` |
| `base_url`         | string | API Base URL                                                         |
| `proxy`            | string | 代理地址，例如 `http://127.0.0.1:7890`                               |
| `api_keys`         | list   | API Keys 列表                                                        |
| `available_models` | list   | 可用模型列表，用於 `/img model` 命令切換                             |

#### 生成行為配置 (`generation`)
| 配置項                 | 型別    | 預設值   | 說明                                                                                                       |
| :--------------------- | :------ | :------- | :--------------------------------------------------------------------------------------------------------- |
| `model`                | string  | `""`     | 當前使用的模型，格式為 `供應商名稱/模型名稱`，不填寫預設使用第一個可用模型，可以透過 `/img model` 指令更新 |
| `timeout`              | number  | `180`    | 請求超時時間（秒）。                                                                                       |
| `max_retry_attempts`   | number  | `5`      | 最大重試次數，可自行設定，最高為 5。                                                                       |
| `max_concurrent_tasks` | number  | `3`      | 最大併發生圖任務數，防止資源耗盡。                                                                         |
| `default_aspect_ratio` | string  | `"自動"` | 預設圖片寬高比。可選值：`自動`、`1:1`、`2:3`、`3:2`、`3:4`、`4:3`、`4:5`、`5:4`、`9:16`、`16:9`、`21:9`。  |
| `default_resolution`   | string  | `"1K"`   | 預設解析度（僅部分模型支援）。可選值：`1K`、`2K`、`4K`。                                                   |
| `show_generation_info` | boolean | `false`  | 生圖成功後展示耗時和生成數量，可於 Web 設定中開啟或關閉。                                                 |
| `show_model_info`      | boolean | `false`  | 生圖成功後展示所使用的模型名稱。                                                                           |

#### 使用者限制配置 (`user_limits`)
| 配置項                    | 型別    | 預設值                       | 說明                                                             |
| :------------------------ | :------ | :--------------------------- | :--------------------------------------------------------------- |
| `umo_blacklist`           | list    | `[]`                         | 會話黑名單，黑名單中的會話將無法使用生圖功能(包含指令和LLM Tool) |
| `blacklist_block_message` | string  | `"❌ 當前會話未啟用生圖功能"` | 當黑名單會話使用生圖功能時返回的文字，留空則不返回任何內容       |
| `rate_limit_seconds`      | number  | `0`                          | 單個使用者兩次請求之間的最小間隔時間（秒），0 表示不限制。         |
| `max_image_size_mb`       | number  | `10`                         | 允許上傳的參考圖最大大小（MB），超過此大小的圖片將被忽略。       |
| `enable_daily_limit`      | boolean | `false`                      | 是否啟用每日生圖數量限制。                                       |
| `daily_limit_count`       | number  | `10`                         | 每個使用者每日允許生成的最大圖片數量。                             |

#### 安全稽核配置 (`safety_audit`)
支援生圖前提示詞稽核和生圖後圖片稽核：
**白名單會話**
| 配置項          | 型別 | 預設值 | 說明                                                                |
| :-------------- | :--- | :----- | :------------------------------------------------------------------ |
| `umo_whitelist` | list | `[]`   | 稽核白名單，白名單中的會話可以跳過稽核流程,使用"sid"指令取得會話umo |
**提示詞稽核 (`prompt_audit`)**
| 配置項            | 型別    | 預設值     | 說明                                                                  |
| :---------------- | :------ | :--------- | :-------------------------------------------------------------------- |
| `blocked_words`   | list    | `[]`       | 自定義遮蔽詞列表，命中任意詞將直接拒絕。                              |
| `enable_ai_audit` | boolean | `false`    | 是否啟用 AI 提示詞稽核。                                              |
| `ai_provider_id`  | string  | `""`       | 稽核使用的 Provider ID。留空時使用當前會話的對話模型 Provider。       |
| `ai_prompt`       | string  | 見預設配置 | AI 稽核提示詞，支援 `{prompt}` 佔位符；未包含時會自動附加當前提示詞。 |

**圖片稽核 (`image_audit`)**
| 配置項            | 型別    | 預設值     | 說明                                                                |
| :---------------- | :------ | :--------- | :------------------------------------------------------------------ |
| `enable_ai_audit` | boolean | `false`    | 是否啟用 AI 圖片稽核。                                              |
| `ai_provider_id`  | string  | `""`       | 稽核使用的 Provider ID。留空時使用當前會話的對話模型 Provider。     |
| `ai_prompt`       | string  | 見預設配置 | AI 稽核提示詞；僅在顯式寫入 `{prompt}` 佔位符時才會附帶當前提示詞。 |

預設情況下，`prompt_audit.ai_prompt` 會把當前生圖提示詞注入到 `{prompt}` 中。
`image_audit.ai_prompt` 預設只稽核圖片本身；只有顯式寫入 `{prompt}` 時才會把當前提示詞一併傳給稽核模型。

#### 快取配置 (`cache`)
| 配置項                   | 型別   | 預設值 | 說明                                       |
| :----------------------- | :----- | :----- | :----------------------------------------- |
| `max_cache_count`        | number | `100`  | 最大快取圖片數量，超過此數量將清理舊圖片。 |
| `cleanup_interval_hours` | number | `24`   | 定時清理間隔（小時）。                     |

#### 預設配置 (`presets`)
| 配置項    | 型別 | 預設值           | 說明                                       |
| :-------- | :--- | :--------------- | :----------------------------------------- |
| `presets` | list | 空列表 | 可自訂的圖片生成預設，詳見下方說明。 |

**預設格式說明：**
- **簡單格式**：`名稱:提示詞`，例如 `動漫風:anime style`
- **高階格式（JSON）**：`名稱:{"prompt":"提示詞","aspect_ratio":"16:9","resolution":"2K","description":"描述"}`

> 本版本不提供任何內建預設，`presets` 預設為空，請依需求自行新增。


### 使用示例

#### 基礎生圖
```
/img gen 一隻在森林裡野餐的兔子
```

#### 使用預設
```
/img preset add 賽博朋克:cyberpunk style, neon lights, futuristic city
/img gen 賽博朋克 貓
```

#### 圖生圖（引用或 @）
1. 傳送或引用一張圖片。
2. 輸入 `/img gen 變成動漫風格[圖片][引用訊息]`。
或者：
`/img gen 變成像素風 @使用者A` (將使用使用者A的頭像作為參考圖)

#### 切換模型
```
/img model        # 檢視列表
/img model 2      # 切換到第二個模型
```
